# PWA Offline Bilty — Implementation Guide

## Overview

The goal is to make the MoveSure frontend a **Progressive Web App (PWA)** that:
- Is **installable** on mobile and desktop (Add to Home Screen / standalone window)
- Is **indexed by search engines** (Next.js/Vite SSR/SSG with correct meta tags)
- Allows staff to **create and view bilties while offline**
- **Syncs** all offline bilties back to the server when the device goes online

---

```json

  "start_url": "/",
  "display": "standalone",

  "theme_color": "#1a56db",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" }
  ]
}
```

Link it in `<head>`:
```html
<link rel="manifest" href="/manifest.json" />
<meta name="theme-color" content="#1a56db" />
<meta name="mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
```

### SEO / Indexing

Add standard meta tags so Google can index the app:

```html
<title>MoveSure — Transport Bilty & Challan Management</title>
<meta name="description" content="Create GR bilties, challans, and trip sheets. Manage consignors, consignees, and fleet from one place." />
<meta name="robots" content="index, follow" />
<link rel="canonical" href="https://app.movesure.in/" />
```

For dynamic bilty pages (`/bilty/:id`) render the GR number, consignor, consignee and route in SSR/SSG so they appear in search results.

## 2. Service Worker

Use **Workbox** (recommended) or a hand-rolled service worker.

### Registration (`src/registerSW.ts`)

```ts
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js').catch(console.error);
  });
}
```

### Caching Strategy (`public/sw.js` or generated by Workbox)

| Resource | Strategy |
|---|---|
| App shell (HTML/JS/CSS) | **Cache First** — serve instantly, update in background |
| Master data (cities, consignors) | **Stale While Revalidate** — fast load + refresh |
| Bilty PDF / images | **Cache First** with expiry |
| API calls (`/v1/bilty`, `/v1/challan`) | **Network First** — fresh data when online, cache fallback |

Workbox config example (`vite.config.ts` with `vite-plugin-pwa`):

```ts
VitePWA({
  registerType: 'autoUpdate',
  workbox: {
    globPatterns: ['**/*.{js,css,html,ico,png,svg}'],
    runtimeCaching: [
      {
        urlPattern: /\/v1\/master\//,
        handler: 'StaleWhileRevalidate',
        options: { cacheName: 'master-data', expiration: { maxAgeSeconds: 86400 } },
      },
      {
        urlPattern: /\/v1\/bilty/,
        handler: 'NetworkFirst',
        options: { cacheName: 'bilty-api', networkTimeoutSeconds: 5 },
      },
    ],
  },
})
```

## 3. Offline Bilty Creation

### Local Storage Layer — IndexedDB

Use **`idb`** (thin Promise wrapper around IndexedDB) to persist offline bilties.

```ts
// src/db/offlineDb.ts
import { openDB } from 'idb';

export const db = await openDB('movesure', 1, {
  upgrade(db) {
    // Pending bilties waiting to be synced
    const store = db.createObjectStore('pending_bilties', { keyPath: 'local_id', autoIncrement: true });
    store.createIndex('synced', 'synced');

    // Cached master data (cities, consignors, consignees, books)
    db.createObjectStore('master_cache', { keyPath: 'key' });
  },
});
```

### Saving a Bilty Offline

When the user submits the bilty form and there is no network:

```ts
// src/services/biltyService.ts
import { db } from '@/db/offlineDb';

export async function submitBilty(payload: BiltyCreate) {
  if (navigator.onLine) {
    return await api.post('/v1/bilty', payload);   // normal path
  }

  // Offline: store locally with a generated temporary GR placeholder
  const local_id = await db.add('pending_bilties', {
    ...payload,
    gr_no: payload.gr_no ?? `OFFLINE-${Date.now()}`,
    synced: false,
    created_locally_at: new Date().toISOString(),
  });
  return { offline: true, local_id };
}
```

> **Note:** For REGULAR bilties, the real GR number must come from `GET /v1/bilty/next-gr`.
> When online, pre-fetch and cache one GR ahead of time.
> When offline, use a provisional `OFFLINE-<timestamp>` label — the real GR is assigned during sync.

## 4. Background Sync

### Using the Background Sync API (supported on Chrome/Android)

Register a sync tag when a bilty is queued:

```ts
// After saving to IndexedDB
const sw = await navigator.serviceWorker.ready;
await sw.sync.register('sync-bilties');
```

In the service worker, handle the `sync` event:

```js
// sw.js
self.addEventListener('sync', (event) => {
  if (event.tag === 'sync-bilties') {
    event.waitUntil(syncPendingBilties());
  }
});

async function syncPendingBilties() {
  const db = await openDB('movesure', 1);
  const pending = await db.getAllFromIndex('pending_bilties', 'synced', false);

  for (const bilty of pending) {
    try {
      // Claim a real GR number first
      const grRes = await fetch('/v1/bilty/next-gr', { headers: authHeaders() });
      const { gr_no, book_id } = await grRes.json();

      const res = await fetch('/v1/bilty', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json', ...authHeaders() },
        body: JSON.stringify({ ...bilty, gr_no, book_id, gr_no_provisional: bilty.gr_no }),
      });

      if (res.ok) {
        await db.put('pending_bilties', { ...bilty, synced: true });
      }
    } catch {
      // Will retry on next sync event
    }
  }
}
```

### Fallback — `online` Event Listener

For browsers that do not support Background Sync (Safari, Firefox):

```ts
// src/sync/onlineSync.ts
window.addEventListener('online', async () => {
  const pending = await db.getAllFromIndex('pending_bilties', 'synced', false);
  for (const bilty of pending) {
    await submitBilty(bilty);  // reuses the same submit function
  }
});
```

## 5. Backend Changes Required

### `POST /v1/bilty` — accept `gr_no_provisional`

Add an optional field `gr_no_provisional` to the bilty schema so the backend can store it for audit / duplicate detection:

```python
# In BiltyCreate (app/v1/bilty.py)
gr_no_provisional: str | None = None   # temporary offline GR, logged for audit
```

Store it in the `bilty` table's `metadata` JSON or add a dedicated column.

### `GET /v1/bilty/next-gr` — pre-reservation (optional optimisation)

Consider an endpoint `POST /v1/bilty/reserve-gr` that claims a GR number but does not yet create a bilty record. The frontend caches the reserved GR in IndexedDB so it can show the correct GR even when offline. The reserved GR is committed when the bilty is eventually synced.

## 6. Offline Indicator UI

Show a banner when the device is offline:

```tsx
// src/components/OfflineBanner.tsx
export function OfflineBanner() {
  const [offline, setOffline] = useState(!navigator.onLine);
  useEffect(() => {
    const on  = () => setOffline(false);
    const off = () => setOffline(true);
    window.addEventListener('online', on);
    window.addEventListener('offline', off);
    return () => { window.removeEventListener('online', on); window.removeEventListener('offline', off); };
  }, []);
  if (!offline) return null;
  return <div className="offline-banner">You are offline — bilties will sync when reconnected</div>;
}
```

Also show a badge/count of unsynced bilties in the navigation.

## 7. Sync Status on Bilty List

| Column | Online bilty | Offline / pending bilty |
|---|---|---|
| GR No | Real GR (e.g. `MUM/0042/25`) | Provisional (`OFFLINE-1746000000000`) |
| Status badge | Normal | `PENDING SYNC` (yellow) |
| Actions | All | View only until synced |

After sync completes, replace the provisional GR with the real one in the local list.

## 8. Master Data Caching (for offline form)

Pre-cache the following so the Create Bilty form works offline:

| Data | Endpoint | IndexedDB key |
|---|---|---|
| Consignors | `GET /v1/master/consignors` | `consignors` |
| Consignees | `GET /v1/master/consignees` | `consignees` |
| Cities | `GET /v1/master/cities` | `cities` |
| Primary bilty book | `GET /v1/bilty-setting/books/primary` | `primary_book` |

Refresh these caches every time the app goes online.
